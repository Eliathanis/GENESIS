# Contributing to GENESIS

Thank you for your interest in contributing to GENESIS! This project aims to bridge MMORPG and tabletop RPG experiences, and we welcome contributions from developers, artists, writers, and tabletop enthusiasts alike.

## 🎯 Project Structure

```
GENESIS/
├── engine/            # Core Godot engine project
├── world_editor/      # Visual world-building tools
├── gm_tools/          # Game Master dashboard and tools
├── ki_integration/    # AI/LLM integration for dynamic content
├── server/           # Multiplayer server infrastructure
├── docs/             # Documentation
└── assets/           # 3D models, textures, sounds
```

## 🚀 How to Contribute

### 1. Reporting Bugs
- Use the GitHub Issues tracker
- Include: Godot version, OS, steps to reproduce, expected vs actual behavior
- Add screenshots/videos if applicable

### 2. Suggesting Features
- Check existing issues first
- Describe the feature in detail
- Explain how it fits the GENESIS vision
- Consider implementation complexity

### 3. Code Contributions
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

### 4. Art & Asset Contributions
- Follow the asset guidelines (coming soon)
- Ensure proper licensing (CC0, CC-BY, or MIT-compatible)
- Include source files when possible

## 💻 Development Setup

### Prerequisites:
- Godot Engine 4.2+
- Git
- (Optional) Python 3.8+ for AI integration tools

### Setup:
```bash
git clone https://github.com/vankinger/GENESIS.git
cd GENESIS/engine
# Open with Godot Editor
```

## 📝 Code Style

### GDScript:
- Follow Godot's official style guide
- Use descriptive variable names
- Comment complex logic
- Keep functions focused and small

### C# (if used):
- Follow .NET naming conventions
- XML documentation for public APIs
- Async/await for I/O operations

## 🧪 Testing
- Write unit tests for critical systems
- Test multiplayer functionality locally
- Verify AI integration responses
- Test world editor usability

## 🎨 Asset Guidelines
- **3D Models**: GLTF 2.0 format preferred
- **Textures**: PNG with power-of-two dimensions
- **Audio**: OGG Vorbis for music/sounds, WAV for UI
- **Icons**: SVG source files when possible

## 🤝 Community Conduct
- Be respectful and inclusive
- Provide constructive feedback
- Help newcomers learn
- Credit contributors appropriately
